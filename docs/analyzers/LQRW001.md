# LQRW001 - GeneratedHashedNamespaceUsageAnalyzer

**Severity:** Warning  
**Category:** Usage  
**Default:** Enabled

## Description
Reports when a `using` directive references a hash-based auto-generated namespace like `Generated_XXXXXXXX`. These namespaces are generated by Linqraft when the `LinqraftNestedDtoUseHashNamespace` option is enabled, and their hash values may change when DTOs are regenerated.

## When It Triggers
When a `using` directive contains a namespace segment matching the pattern `Generated_XXXXXXXX` where `XXXXXXXX` is an alphanumeric hash of at least 8 characters.

## Why This Is A Problem
Auto-generated namespaces with hash suffixes are implementation details that may change:
- When the source code structure changes
- When DTO properties are modified
- When regenerating the code with different options

Explicitly depending on these namespaces can lead to compilation errors when the DTOs are regenerated.

## How To Fix
Instead of using the hash-based namespace directly, consider:
1. Using explicit DTO types with the `SelectExpr<TResult>` overload
2. Using `var` for variable declarations to avoid explicit type references
3. Defining your own DTO types manually

## Example
Before:
```csharp
using MyApp.Generated_A1470000;  // LQRW001

class Test
{
    void Method()
    {
        ItemsDto dto = ...;  // Explicitly uses auto-generated type
    }
}
```

After:
```csharp
// Define explicit DTO type or use var
class Test
{
    void Method()
    {
        var dto = ...;  // Uses var - no warning
    }
}
```
